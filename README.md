# Мишаня шаманит

MVP-вебка на Go 1.25+ для абсурдной киберпанк-деревенской игры в угадайку по рисунку.
Один игрок получает случайную фразу вроде `дед кибер-пасечник на дроновом тракторе`, рисует её на canvas, остальные смотрят рисунок в реальном времени и кидают догадки в чат. Когда кто-то угадал — рисующий подтверждает победителя.

## Что теперь умеет

- создание комнаты с коротким shareable-кодом
- вход по ссылке `/room/<CODE>` или по коду
- live-обновления через SSE
- общая доска для рисования в реальном времени
- чат с догадками
- случайные абсурдные кибер-деревенские фразы
- таймер раунда
- подтверждение победителя рисующим
- ротация роли рисующего после удачного угадывания
- docker-first запуск без внешних сервисов

## Ограничения MVP

- состояние хранится только в памяти процесса; рестарт контейнера = комнаты пропадают
- нет авторизации и reconnect через полноценные сессии; используется локальный `playerId` в браузере
- стирание холста глобально не синхронизировано отдельной командой; новый раунд очищает холст полностью
- проверка угадывания ручная: победителя подтверждает рисующий
- оптимизировано прежде всего под 2 игроков, но 3+ тоже могут заходить и угадывать

## Архитектура

```text
cmd/server/main.go         entrypoint
internal/config            env-конфиг
internal/httpx             HTML + JSON API + SSE
internal/service           in-memory room/game state
web/templates              страница лобби и игры
web/static                 фронт, canvas, chat, realtime UI
```

## Быстрый запуск

```bash
cp .env.example .env
docker compose up --build
```

Потом открывай:

- http://localhost:8080

## Игровой поток

1. Первый игрок создаёт комнату и получает код/ссылку.
2. Второй игрок заходит по ссылке.
3. Рисующий жмёт «Новый раунд».
4. Сервер выдаёт только рисующему секретную фразу.
5. Остальные видят маску фразы и следят за рисунком.
6. Догадки идут в чат.
7. Когда кто-то угадал — рисующий подтверждает игрока кнопкой.
8. Сервер объявляет победителя и передаёт право рисовать ему.

## ENV

- `APP_NAME` — имя приложения
- `APP_DOMAIN` — декоративный домен в UI
- `HTTP_ADDR` — адрес HTTP-сервера
- `PUBLIC_BASE_URL` — база для shareable-ссылок
- `ROUND_DURATION_SECONDS` — длина раунда, по умолчанию `90`
- `ROOM_CODE_LENGTH` — длина кода комнаты, по умолчанию `6`
- `MAX_CHAT_MESSAGES` — сколько сообщений держать в памяти на комнату
- `MAX_BODY_BYTES` — лимит размера JSON-запроса

## API, если коротко

- `POST /api/rooms` — создать комнату
- `POST /api/rooms/join` — войти в комнату
- `GET /api/rooms/{code}?playerId=...` — получить состояние комнаты
- `GET /api/rooms/{code}/events?playerId=...` — SSE-поток обновлений
- `POST /api/rooms/{code}/start` — старт раунда
- `POST /api/rooms/{code}/guess` — отправить сообщение/догадку
- `POST /api/rooms/{code}/stroke` — отправить штрих рисунка
- `POST /api/rooms/{code}/confirm` — подтвердить победителя
